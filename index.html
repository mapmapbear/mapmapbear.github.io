1.交叉编译

什么是交叉编译?

指一个在某个系统下可以产生另一个系统的可执行文件编译过程。交叉编译在目标系统平台（开发出来的应用程序序所运行的平台）难以或不容易编译时非常有用

2.开始编译 (๑•̀ㅂ•́)و✧

选择mingw在Linux下的编译器

一般Linux to Windows用的是X86_64-w64-mingw32-g++(X86平台)

一个简单的例子: main.cpp -> main.exe

如果Windows上没有安装c++的编译器的话,像libstdc++-6.dll等常见的动态链接库是没有的哦

要把Mingw安装在系统中,并且把动态链接库所在目录插入环境变量中,就好像玩单机游戏时候装的VC++运行库一样(玩游戏必不可少!)

LeetCode 202.快乐数

1.题目解释

编写一个算法来判断一个数 n 是不是快乐数。

「快乐数」定义为：

对于一个正整数，每一次将该数替换为它每个位置上的数字的平方和。
然后重复这个过程直到这个数变为 1，也可能是 无限循环 但始终变不到 1。
如果 可以变为 1，那么这个数就是快乐数。
如果 n 是快乐数就返回 true ；不是，则返回 false 。

2.思路解释

具体的快乐数的含义就是每位的平方和,最后得到值为1的即为快乐数

(快慢指针) O(1)

这道题其实就是包装后的环形链表，其中
慢指针就是每次把数字各位求一次平方和
快指针就是每次把数字各位求两次平方和
有两种情况
是快乐数：最后两个指针都会变成1，重合在一起
不是快乐数：相当于一个环，快慢指针总会在某个数处相遇
即：不管是不是快乐数，快慢指针最终都会汇聚到一个数上，我们只需要判断这个数是不是1即可
